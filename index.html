THE SOLUTION

The iSuara Bridge

How It Works

1. Input

A standard webcam captures the user's continuous BIM gestures in real-time, requiring no specialized gloves or hardware.

2. Processing

An edge-optimized LSTM model extracts skeletal movements, while a cloud-based semantic layer pieces the raw signs into context.

3. Output

Fluid, grammatically correct Malay sentences are rendered instantly as live subtitles on the recipient's screen.

Key Capabilities

Real-Time Edge Inference

Built for speed. By running our custom LSTM model natively on the edge via TensorFlow Lite, we achieve ultra-low latency inference, ensuring conversations feel natural and uninterrupted.

Semantic Contextualization

Sign language syntax doesn't map 1:1 to spoken languages. Our integration with Gemini 2.5 Flash acts as a semantic bridge, transforming raw, disjointed glosses into perfectly structured Malay sentences.

Hardware Agnostic & Private

No expensive depth cameras or proprietary wearable gloves required. iSuara runs entirely off a standard 2D laptop webcam, making it universally accessible while processing skeletal data locally to preserve user privacy.

Academic & Professional Focus

Unlike basic consumer translation apps, our vocabulary bank is specifically curated for tertiary education and corporate environments, allowing DHH users to participate in complex STEM discussions.

WORKFLOW

System Implementation

The real-time, 5-stage pipeline powering iSuara's dual-layer architecture.

1

Video Ingestion

  • OpenCV video capture runs locally.
  • Captures steady 30fps stream.
  • Stores rolling 30-frame memory buffer.
2

Skeletal Extraction

  • MediaPipe maps 3D hand/pose nodes.
  • Compiles 258-point dense vector.
  • Normalizes to shoulder & applies EMA smoothing.
3

LSTM Inference

  • Feeds optimized TensorFlow Lite edge model.
  • Evaluates temporal motion probability.
  • Triggers on >60% confidence + 10-frame cooldown.
4

Semantic Context

  • Async threading prevents UI freeze.
  • Gemini 2.5 Flash API maps glosses to Malay.
5

Execution & Output

  • Receives parsed Malay payload.
  • Re-syncs text to main execution thread.
  • Renders live OpenCV subtitles overlay.

Iteration 1

Initial Challenges

Technical Performance

  • Camera tracking struggled with natural, rapid hand movements (latency).
  • Model failed to differentiate "minimal pairs" (similar signs like "Kakak" vs "Abang").
  • High environmental sensitivity; accuracy dropped when standing further away.

UX & Vocabulary

  • Severe vocabulary limitations (missing common words like "Makan").
  • Onboarding gap: "Green light" indicator was not intuitive for new users.
Iteration 2

Targeted Solutions

Architecture Upgrades

  • Parallel Processing & 60fps: Split tasks between CPU and GPU to achieve a smooth 50fps, eliminating camera lag to easily catch rapid gestures.
  • Dynamic Hand Cropping: Instantly track wrist coordinates to focus only on the hands, ensuring high accuracy even if the user stands far away.
  • Continuous Multi-Window Inference: Analyze back-to-back frames dynamically to prevent missed gestures and accurately differentiate visually similar signs.
  • EMA Smoothing: Exponential Moving Average (EMA) smoothing filters out single-frame visual noise so the model isn't tricked by a random anomaly.

Challenges Faced

Dataset Inconsistency

BIM Sign Bank videos lack standardized framing (varying between hand-only and half-body shots) and utilize a single camera angle, complicating spatial tracking.

Compute & Time Trade-offs

Training a model on the entire 2,966-word BIM lexicon consumed a lots of time. We had to downsample to a curated subset of high-frequency academic terms.

Model Hallucinations

The current sequence-prediction model occasionally misclassifies ambiguous gestures, resulting in incorrect raw word outputs before passing through the GenAI semantic layer.
